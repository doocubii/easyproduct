# 변경 내역 (Changelog)

이 저장소(easyproduct 스킬 세트)의 버전별 변경 내역이다. 버전·날짜를 언제 확정하는지는 `VERSIONING.md`를 따른다.
형식은 [Keep a Changelog](https://keepachangelog.com/ko/) 관례를 참고하되 한국어로 정리한다.

- **버전 번호**는 작업 브랜치가 `beta`에 머지될 때 확정된다.
- **릴리즈 날짜**는 단일 필드로, 머지될 때마다 기록된다 — `beta` 머지 시 beta 머지일, `main` 머지 시 그 날짜로 갱신.

## [0.4.3] — 2026-07-20 (beta)

> 상태: `beta` 머지로 **버전 0.4.3 확정**, 릴리즈 날짜 = beta 머지일(2026-07-20).
> 설치 스크립트를 컨펌형으로 — 전역/프로젝트 선택을 대화로.

### Changed (변경)
- **`install.sh`·`install.ps1`: 인자 없이 실행하면 설치 위치를 컨펌한다.** "전역(~/.claude)에 설치할까요? [Y/n]"
  → Y/엔터=전역(`~/.claude/skills`), n=프로젝트 폴더 경로를 입력받아 `<폴더>/.claude/skills`에 설치.
  `-y`(ps1은 `-Yes`)면 컨펌 없이 전역. 위치 인자(`./install.sh <폴더>`·`-Base <폴더>`)는 그대로 비대화 설치(`.`=현재 폴더).
  비대화 입력(파이프 등)이면 컨펌을 건너뛰고 전역으로 폴백(안내 출력).
- README 설치 절을 컨펌 동작(+ `-y`)에 맞게 갱신.

## [0.4.0] — 2026-07-20 (beta)

> 상태: `beta` 머지로 **버전 0.4.0 확정**, 릴리즈 날짜 = beta 머지일(2026-07-20).
> 화면 설계 기계 블록 구조 재설계 + 스킬 부합 린트 신설(0.x에서 구조 변경 포함은 MINOR).

### Changed (변경)
- **화면 설계 `screendesign.screens`의 데이터 표현 재설계.** 평탄 `data:[…]`를 방향별로 나눠
  `data{ display(보여주는/읽기), io[{action,ui?,op?,sends[],receives[]}](동작별 보냄/받음), bindings[{ui,vars[]}](요소↔변수) }`로.
  손실 미러를 없애고 화면↔데이터·화면↔API(오퍼레이션) 계약 점검의 근거 마련(draft라 v1 제자리 정제).
  스키마·SKILL·suite Stage 4·`check-docs.mjs`·도메인 템플릿을 함께 갱신. `check-docs`는 구 평탄 `data` 호환.

### Added (추가)
- **스킬 부합 린트 `scripts/skill-lint.mjs`(무의존 Node).** 베타 머지 게이트의 기계 가능 부합 항목
  (빈 껍데기 금지·뜻 필수·정합성 점검 절 존재·점검기 자산)을 자동 점검, 실패 시 종료코드 1.

### Docs (문서)
- **8개 스킬 "정합성 점검" 절을 점검 3층 계약에 정렬**(누가·무엇을·언제 + ssot 원본 기준 재생성). `checker-guide`에 점검 3층 원칙·스킬 부합 게이트 명문화.
- **CLAUDE.md: "대대적 업그레이드 후 전체 흐름 도그푸드(원래 목적 회귀 점검)" 규칙** 추가. 이번 화면 data 재설계를 이 규칙대로 도그푸드해 통과 확인(신규 세트 생성 → check-docs·skill-lint 통과).

## [0.3.0] — 2026-07-18 (beta)

> 상태: `beta` 머지로 **버전 0.3.0 확정**, 릴리즈 날짜 = beta 머지일(2026-07-18).
> 릴리즈 날짜는 단일 필드이며, `main` 머지(정식 릴리즈) 시 그 날짜로 갱신된다.

### Added (추가)
- **문서 기계 판독 층 도입** — 각 스킬 산출 문서를 "사람용 + 기계용 2층"으로 확장. 소프트웨어(하네스·CI)가 규약으로 스펙을 조회·점검할 수 있다.
  - 문서 맨 앞 **frontmatter**(`doc_type`·`version`·`ssot`·`machine.*`)로 문서 종류와 기계 블록 위치·검증 계약을 선언.
  - 태그된 **기계 블록**: 데이터 모델 `datamodel.group`(`DATA.*`), IA `ia.features`(`FEAT.*`), 정책서 `policy.rules`(`POL.*`), 화면 설계 `screendesign.screens` + 인벤토리 `uicomponents.list`(`UI.*`), 디자인 컨셉 `design.tokens`, 시나리오 `scenario.trace`(`SCN.*`), 문서 묶음 색인 `docbundle.docs`(세트 매니페스트).
  - 기획서·설계서·유저 스토리는 상위 요구 문서라 **frontmatter(식별)만** 두고 기계 블록은 없음(그 내용의 ID는 하위 문서가 확정).
- **payload 스키마**(`schemas/*.v1.schema.json`) — 각 기계 블록의 JSON Schema 계약. 스킬 소유 고정 자산을 산출물 옆 `schemas/`에 버전 핀으로 복사.
- **크로스도큐먼트 참조 네임스페이스** — 접두사(`FEAT./DATA./POL./UI./SCN./token`)가 곧 "어느 등기부에서 조회할지"의 타입 표시. 문서 간 참조 무결성(죽은 링크)을 기계가 점검.
- **세트 전역 기계 점검 게이트**(suite Stage 4) — 시나리오 유무와 무관하게 각 문서의 스키마 검증 + 크로스도큐먼트 참조 무결성을 점검.
- **점검자 개발 가이드** `easyproduct-suite/references/checker-guide.md` — frontmatter 필드 의미·doc_type/anchor 레지스트리·참조 라우팅·점검자 알고리즘.
- **무의존 참조 점검기** `easyproduct-suite/scripts/check-docs.mjs`(Node) — suite가 문서 세트에 겨눠 실행. 프로젝트로 복사하지 않고 소스는 요청 시 제공.
- 각 스킬에 **옛 문서 gap-fill 업그레이드 경로** — frontmatter·기계 블록·스키마가 없는 기존 문서를 표·산문은 그대로 두고 기계층만 보강.

### Changed (변경)
- "봉투" 표기를 md 생태계 표준어 **frontmatter**로 통일(YAML 키·블록·스키마는 불변).

- **문서 정합성** — 시나리오 도입 게이트를 "IA 이상"으로 확장, README 흐름도 순서(기능 관계도→시나리오)·명명 규칙(SCN.*/UC.*)·suite 설명 정정.
