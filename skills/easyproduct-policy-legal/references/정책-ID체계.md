# 정책 ID 체계 (`POL.*`) — 참조 가능한 안정 ID

## 왜 필요한가

정책서의 각 운영 규칙에 **안정적이고 참조 가능한 ID**를 부여한다. 다른 문서(시나리오·화면 설계서·유스케이스)가 "이 흐름은 이 규칙을 따른다"고 **이름이 아니라 ID로** 가리킬 수 있게 하기 위해서다. 규칙 제목의 표현이 바뀌어도(예: "환불·취소 기준" → "환불 규정") ID(`POL.payment.refund`)는 그대로라 링크가 끊어지지 않는다.

이 체계는 기능 ID `FEAT.도메인.기능명`, UI 컴포넌트 ID `UI.카테고리.컴포넌트명`, 데이터 변수 `그룹.필드`와 나란한 **정책의 원본(SSOT) ID**다.

## 형식

```
POL.{도메인}.{규칙명}
```

- **도메인**: 정책 섹션의 종류(아래 표준 도메인).
- **규칙명**: 그 섹션 안의 개별 규칙 단위. camelCase 또는 짧은 영단어.
- 예: `POL.member.suspend`, `POL.payment.refund`, `POL.noshow.cancel`.

한 섹션 전체를 가리킬 때는 도메인까지만: `POL.payment.*` (예: "결제 관련 정책 전체").

## 표준 도메인 · 규칙 매핑

정책서 템플릿의 섹션과 1:1로 맞춘다. **해당되는 섹션만** 문서에 들어가듯, ID도 넣은 규칙에만 부여한다.

| 정책 섹션 | 도메인 | 대표 규칙 ID |
|---|---|---|
| 1. 회원 정책 | `member` | `POL.member.join`(가입 자격) · `POL.member.grade`(등급) · `POL.member.withdraw`(탈퇴) · `POL.member.suspend`(이용 제한·정지) |
| 2. 콘텐츠·게시물 | `content` | `POL.content.allowed`(허용/금지) · `POL.content.copyright`(저작권·권리) · `POL.content.takedown`(삭제·블라인드) |
| 3. 결제·환불·취소 | `payment` | `POL.payment.charge`(결제 수단·시점) · `POL.payment.refund`(환불·취소 기준) · `POL.payment.subscription`(정기결제·구독) |
| 4. 신고·제재 | `report` | `POL.report.target`(신고 대상) · `POL.report.process`(처리 절차) · `POL.report.sanction`(제재 단계) |
| 5. 노쇼·취소 | `noshow` | `POL.noshow.cancel`(취소 시점·방법) · `POL.noshow.penalty`(노쇼 처리) · `POL.noshow.host`(주최자·참여자 책임) |
| 6. 위치정보 이용 | `location` | `POL.location.use`(용도) · `POL.location.toggle`(켜고 끄기) |
| 7. 분쟁 처리 | `dispute` | `POL.dispute.process`(분쟁 처리 원칙) |

새 규칙을 추가할 때는 위 도메인 안에서 규칙명을 새로 짓는다. 표에 없는 성격의 정책 섹션이 생기면 새 도메인을 만든다(짧은 영단어).

## 규칙

1. **ID는 규칙의 원본(SSOT)이다.** 정책서가 `POL.*` ID를 정하고, 다른 문서는 이 ID를 **참조만** 한다(베끼지 않는다).
2. **한 번 부여한 ID는 바꾸지 않는다.** 제목·문구를 고쳐도 ID는 유지한다. 규칙을 **삭제**하면 그 ID도 폐기하고, 그 ID를 참조하던 문서(시나리오 등)는 재점검 대상이 된다(죽은 링크).
3. **재진입 부분 수정 시 ID 보존**: 지정되지 않은 규칙의 ID는 글자 그대로 유지한다. 조항을 켜고 끌 때 딸린 ID도 함께 정리한다.
4. **표시 위치**: 정책서 문서에서 각 규칙 제목 옆에 인라인 코드로 ID를 붙인다. 예: `**환불·취소 기준** \`POL.payment.refund\``. 섹션 제목에는 도메인을 붙인다: `## 3. 결제·환불·취소 정책 \`POL.payment.*\``.

## 참조하는 쪽에서 쓰는 법

시나리오·유스케이스·화면 설계서가 어떤 결정이 정책에 근거함을 표시할 때, **`이름(ID)`** 형식으로 적는다(사람이 읽는 이름 + ID를 함께):

```
When  무료 한도를 넘겨 검토 실행을 시도  → 결제 정책(POL.payment.charge)
Then  결제/견적 경로로 유도               → 결제(FEAT.billing.checkout) · 결제 정책(POL.payment.*)
```

- 참조는 **가리키기만** 한다 — 정책 문장·수치를 시나리오에 베껴 두지 않는다. 정책이 바뀌면 참조하는 문서를 재도출한다(링크-온리 원칙).
- `이름(ID)` 표기는 **새로 만들거나 수정하는 문서에만** 적용한다. 기존 문서가 이 형식이 아니어도 틀린 것이 아니므로 억지로 바꾸지 않는다.
