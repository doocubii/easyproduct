# 디자인 컨셉: [프로젝트 이름]

> 이 문서는 "이 서비스가 **어떤 색·글꼴·느낌으로 보일지**"를 사람이 읽기 쉽게 정리한 것입니다.
> **디자인 안을 3가지 담았습니다.** 셋 다 이 문서 안에 있고, 지금 선택된 안은 아래 "선택된 안"에 적혀 있습니다.
> 각 안의 색·버튼·샘플 화면은 함께 드린 미리보기 이미지(PNG)와 `디자인안-비교.html`에서 눈으로 비교할 수 있습니다.
> 색상 코드(예: #2563EB) 같은 값은 나중에 만들 사람(또는 AI)이 그대로 쓸 수 있게 적어둔 것입니다.

## ✅ 선택된 안  ← 여기만 바꾸면 선택이 바뀝니다
- **선택된 안**: **1안 [안 이름]**
- **왜 이 안인지**: (대상·유형·이미 정해진 색 등을 근거로 한두 줄)
- 다른 안으로 바꾸고 싶으면: "2안으로 해줘"처럼 말씀만 주세요. 위 한 줄만 고치면 됩니다.
- 아예 새로운 방향(예: "더 어둡고 고급스럽게")을 원하시면, 세 안을 새로 만들어 드립니다.

> ⚠️ **샘플 페이지·미리보기 이미지는 "선택을 돕기 위한 임시 참고물"입니다.**
> 개발이 따라야 할 **표준·근거는 이 문서에 적힌 값**(색상 코드·글꼴·컴포넌트 규칙)입니다.
> 샘플 화면의 **레이아웃·배치를 표준으로 삼지 마세요.** 실제 화면 구성은 화면 구조(IA)와 다음 단계에서 정합니다.
> 안이 새 방향으로 바뀌면 이전 샘플은 폐기됩니다.

## 한 줄 요약
누구를 위한 무엇이고, 한마디로 어떤 느낌인지. (예: "동네 모임을 위한, 따뜻하고 편안한 느낌의 앱")

## 공통으로 지키는 것 (모든 안에 적용)
### 접근성
- 글자와 배경의 대비를 충분히 (잘 보이게).
- 색만으로 정보를 구분하지 않기 (아이콘·글자도 함께).
- 버튼·누르는 곳을 충분히 크게.
- (특별한 대상이 있으면) 글자 더 크게 / 대비 더 강하게.

### 자동으로 넣은 것 / 가정 / 아직 안 정한 것
- 자동으로 넣은 것: (AI가 분위기를 보고 정한 것)
- 가정: (사용자가 말하지 않아 임의로 정한 것)
- 아직 안 정한 것: [확인 필요: …]

### 글꼴
- 제목용 / 본문용 (예: Pretendard Bold / Regular) — 왜 이 느낌에 맞는지 한 줄.
- 크기 위계: 큰제목 / 제목 / 소제목 / 본문 / 작은 글씨.

### 모양·형태
- 모서리(각짐↔둥긂) / 그림자(없음↔뚜렷) / 여백·밀도(빽빽↔여유) — 값 + 느낌.

### 컴포넌트 스타일
- 버튼(기본·보조·비활성) / 입력창(기본·포커스·오류) / 카드.

### 아이콘·이미지 · 톤앤매너 · 다크모드
- 아이콘(선형↔채움) / 이미지(사진↔일러스트).
- 말투: 버튼·안내·오류 예시 문구.
- 다크모드: 지원 여부 + 어두운 바탕 색 처리.

### 참고 레퍼런스 / 하지 말 것
- 참고: (무엇의 어떤 점을 참고했는지)
- 하지 말 것: (이 안에서 피할 방향)

## 참고: 안이 바뀔 때
- **안 중에서 고르기**(예: "2안으로") → 위 "선택된 안" 한 줄만 바꿉니다. 세 안 내용과 샘플은 그대로 둡니다.
- **새 방향 지시**(예: "더 따뜻하게 다시") → 세 안과 샘플을 폐기하고 새로 3안·새 샘플을 만듭니다.

## (기계 표현) 디자인 토큰 (선택된 안)

**선택된 안**의 색·모서리·여백·타이포 값을 **소프트웨어가 읽는 사본**으로 옮긴 것입니다. 위 안별 정의(사람용)가 원본이고, 선택이 바뀌거나 값이 바뀌면 이 블록도 함께 갱신합니다. 화면 설계서·UI 인벤토리가 이 토큰을 이름(`color.primary`·`spacing.md` 등)으로 참조합니다.

```json design.tokens
{
  "selected": "1",
  "tokens": {
    "color": {
      "primary": "#2563EB",
      "background": "#FFFFFF",
      "textBody": "#1F2937",
      "success": "#16A34A",
      "error": "#DC2626"
    },
    "radius":  { "sm": "8px", "md": "12px", "lg": "16px" },
    "spacing": { "sm": "8px", "md": "16px", "lg": "24px" },
    "type":    { "title": "24px/700", "body": "16px/400", "caption": "13px/400" }
  }
}
```

- `selected` = 위 "선택된 안"과 같은 번호. `tokens.color.*` = hex 색, `radius`/`spacing`/`type` = 이름별 값.
- 위 값은 형식 안내용입니다 — 실제로는 **선택된 안의 실제 값**을 담습니다(다른 안으로 바뀌면 그 안의 값으로 다시 만듭니다).

### 이 블록으로 소프트웨어가 자동 점검할 수 있는 것

이 토큰 사본은 사람이 아니라 소프트웨어가 읽는 것이라, 아래를 자동으로 확인할 수 있습니다(직접 하실 일은 아니고, 참고용입니다).

- **형식 검증** — 색 값이 올바른 색상 코드(hex) 꼴인지, `selected`가 있는지.
- **정의 ↔ 토큰 정합** — 선택된 안의 색·모양 값과 이 블록이 서로 어긋나지 않는지.
- **다른 문서와의 참조** — 화면 설계서·UI 인벤토리가 가리키는 토큰(`color.primary`·`spacing.md` 등)이 실제로 여기 있는지(끊긴 토큰 참조 적발).
