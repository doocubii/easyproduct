# easyproduct-data-model

> 개발 지식이 없는 사람도 대화만으로, 서비스가 다룰 데이터 모델(데이터 사전)을 만들고·고치고·찾아보게 돕는 스킬입니다. "회원·요청·거래가 있어" 같은 정보 덩어리만 알면 각 덩어리에 표준 필드를 자동으로 깔아, 하나의 데이터 모델 문서(md)를 통째로 초안으로 잡아줍니다.

[easyproduct 스킬 세트](../../README.md)의 일부입니다.

## 무엇을 하나

- 정보 덩어리(회원·요청·거래 등)에서 기본 데이터 모델을 추출해 **전체 초안을 한 번에** 문서로 만듭니다. 매번 묻지 않고 먼저 만든 뒤 고치게 합니다.
- 모든 덩어리에 **공통 관리 필드**(`id`·`createdAt`·`updatedAt` 등)를, 회원·거래계 핵심 종류에는 **표준 필드 프리셋**을 자동으로 깝니다. 자동으로 넣은 건 표시해 빼기 쉽게 합니다.
- 변수명은 스킬이 **`그룹.필드`** 꼴(그룹=영문 소문자 단수, 필드=camelCase)로 지어 줍니다. 사용자가 영어를 직접 짓지 않습니다.
- 데이터 모델 문서를 기준선(SSOT)으로 필드를 추가·삭제·이름변경·설명수정하고, 특정 데이터·필드를 찾아 답합니다.
- 각 그룹 표 **바로 아래에 그 구조를 담은 JSON 미러**를 함께 넣어 소프트웨어가 바로 읽게 합니다. **표가 원본, JSON은 미러**라 표를 고치면 JSON을 다시 생성합니다(손으로 고치지 않음).
- 요청하면 그 인라인 JSON 미러들을 **서비스 단위 한 파일(`.json`)로 묶어** 내려줍니다.

## 언제 쓰나

- "데이터 모델", "데이터 사전", "데이터 구조", "어떤 데이터가 필요하지", "필드/항목 정리"를 만들거나 고치거나 찾아보려 할 때.
- "회원에 뭐가 들어가지?", "이 서비스가 다루는 정보가 뭐지?", "금액 관련 데이터 뭐 있어?"처럼 정보 항목이 막막하거나 물어볼 때.
- 기존 데이터 모델 파일(md)을 가져와 필드를 추가·수정·조회하려 할 때, 또는 "json으로 뽑아줘"처럼 기계가 읽기 좋게 변환하려 할 때.
- 기획서·화면 구조(IA)는 있는데 그 안의 정보 항목을 데이터로 정리하려 할 때.

## 산출물

- **`데이터모델-[서비스].md`** — 데이터 모델 문서이자 원본(SSOT). 그룹별 표(원본) + 표 아래 JSON 미러를 함께 담습니다. (상시 산출물)
- **`데이터모델-[서비스]-추출.json`** — 요청 시에만 인라인 미러들을 한 파일로 묶은 사본. 이 파일을 고쳐도 md 원본에 반영되지 않습니다.

## 단독 실행 vs 세트

- 기본은 **단독 동작**입니다. 기획서가 없어도 정보 덩어리(파일이나 말)만 받으면 새 데이터 모델을 만들 수 있습니다.
- 데이터 모델 파일(md)을 받으면 그것을 기준선(SSOT)으로 삼아 조회·수정·추가를 이어가고, 새로 생성하지 않습니다. 기획서를 함께 받아도 데이터 모델이 우선이며 기획서 기준으로 덮어쓰지 않습니다.
- 누가 이 스킬을 어떻게 가져다 쓰는지는 이 스킬의 관심사가 아닙니다 — 연계는 쓰는 쪽이 정하고, 요청 처리 규칙·결과 회신은 특정 스킬과의 계약이 아니라 입력 요건·출력 형식일 뿐입니다.

## 구성 파일

- `SKILL.md` — 스킬 본문(생성·수정·조회·json 추출, 명명 규칙, 요청 처리·정합성 점검 규칙 등).
- `assets/data-model-template.md` — 데이터 모델 문서(원본) 구조. 맨 앞 SSOT·수정 원칙 문구 포함.
- `references/standard-fields-preset.md` — 공통 관리 필드 + 회원·거래계 5~7종 표준 필드 지식베이스. 생성·등록 시 참고.

## 세트 안에서의 위치

- 기획서(easyproduct-doc-builder)나 화면 구조(easyproduct-ia-designer)를 만든 뒤, 그 안의 정보 항목을 데이터로 정리하는 단계입니다.
- 데이터의 원본(SSOT)은 이 스킬이 만드는 데이터 모델 문서 하나입니다. 화면·코드 등 다른 곳은 이 문서의 변수명을 이어받아 참조하고, 데이터를 바꿀 때는 이 스킬에서 바꿉니다.
- 읽기(read)는 데이터 모델 문서(md)를 직접 읽으면 되고, 쓰기·판정(write/resolve)이 필요할 때만 이 스킬을 부릅니다.
