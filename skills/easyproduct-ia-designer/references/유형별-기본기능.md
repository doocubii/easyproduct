# 유형별 기본 기능 지식베이스

이 파일은 **서비스 유형만 알면 자동으로 붙여야 하는 기능 목록**을 담고 있다.
초안을 잡을 때 이 목록을 꺼내 **사용자가 말하지 않아도** 채워 넣는다.
"이건 당연히 필요하겠죠?" 하고 매번 묻지 말고, **일단 초안에 넣고 표시**한 뒤 나중에 빼게 한다.

사용 방법:
1. Step 1에서 받은 유형으로 아래 **해당 유형**을 찾는다.
2. **공통 기능**(모든 서비스) + **유형 기능** + **백오피스 기능**을 합쳐 초안 목록을 만든다.
3. 사용자가 말한 **핵심 기능**을 맨 위에 두고, 자동으로 넣은 것은 "기본(자동)"으로 구분 표시한다.
4. 유형이 애매하거나 여러 개에 걸치면 가장 가까운 것 하나를 고르되, 걸치는 다른 유형의 기능도 참고해 얹는다.

---

## 공통 기능 (거의 모든 서비스에 자동으로 넣는다)

### 사용자(프론트)
- **홈/메인** — 처음 들어왔을 때 보는 화면
- **회원가입 / 로그인** — 이메일 가입, 소셜 로그인(카카오·구글 등), 비밀번호 찾기
- **마이페이지 / 내 정보** — 내 활동·설정을 모아 보는 곳
- **설정** — 알림, 비밀번호 변경, 회원 탈퇴
- **알림** — 나에게 온 소식(선택적이지만 대부분 필요)
- **검색** — 서비스에 정보가 쌓이면 거의 항상 필요
- **고객센터 / 1:1 문의** — 문의 남기기
- **FAQ / 자주 묻는 질문**
- **공지사항**
- **이용약관** (법적 필수)
- **개인정보처리방침** (법적 필수)

> 한국 서비스 참고: 결제가 있으면 **전자상거래 표시·환불 정책**, 마케팅 알림을 보내면 **마케팅 수신 동의**, 위치를 쓰면 **위치정보 이용 동의**가 추가로 필요하다. 초안에 함께 표시한다.

### 관리자(백오피스) — 거의 모든 관리자 화면에 자동으로 넣는다
- **관리자 로그인**
- **대시보드** — 핵심 숫자를 한눈에(가입자, 오늘 활동 등)
- **회원 관리** — 목록·검색·상세·정지/탈퇴 처리
- **문의(CS) 관리** — 들어온 문의에 답하기
- **공지 / FAQ / 약관 관리** — 프론트에 보이는 안내글 편집
- **관리자 계정·권한 관리** — 누가 무엇을 할 수 있는지

## 유형별 기능

각 유형은 **대표 예시 → 사용자 기능 → 백오피스 기능** 순서다.
백오피스는 "프론트에서 생기는 것을 누군가 관리해야 한다"는 원칙으로 짝을 맞춘다.

### 1. 커뮤니티 / 게시판
예시: 동호회 게시판, 취미 커뮤니티, Q&A 사이트
- 사용자: 글 목록·상세, 글쓰기·수정·삭제, 댓글, 좋아요/추천, 카테고리·태그, 신고, 인기글, 프로필
- 백오피스: 게시글·댓글 관리(블라인드/삭제), **신고 처리**, 카테고리 관리, 회원 활동 통계, 금칙어·스팸 관리

### 2. 이커머스 / 쇼핑몰
예시: 온라인 상점, 브랜드몰
- 사용자: 상품 목록·상세, 검색·필터·정렬, 장바구니, 주문·결제, 배송 조회, 주문 내역, 리뷰·평점, 위시리스트, 쿠폰·적립금
- 백오피스: **상품 관리**(등록·재고), **주문 관리**(상태·배송), **정산·매출 통계**, 쿠폰·프로모션 관리, 리뷰 관리, 반품·교환·환불 처리, CS 관리

### 3. 예약 / 부킹
예시: 숙박·식당·시설·클래스 예약
- 사용자: 검색(날짜·인원·조건), 대상 목록·상세, 가용성 확인, 예약·결제, 예약 내역·취소, 리뷰
- 백오피스: **예약 관리**(승인·거절·현황), **가용성/캘린더 관리**, 대상(상품·시설) 관리, 정산, 노쇼·취소 정책 관리, 통계

### 4. SaaS / 업무 도구
예시: 협업 툴, 관리 프로그램, 대시보드형 서비스
- 사용자: 온보딩, 대시보드, 핵심 작업 화면, 팀·조직 만들기, 팀원 초대·권한, 요금제·구독, 결제, 설정
- 백오피스: 조직·워크스페이스 관리, **구독·결제 관리**, 사용량 모니터링, 기능 플래그, 지원 요청 관리, 통계

### 5. 콘텐츠 / 미디어
예시: 블로그, 뉴스, 매거진, 영상 플랫폼
- 사용자: 콘텐츠 목록·상세, 카테고리, 검색, 구독·북마크, 댓글, 추천/관련 콘텐츠, 공유
- 백오피스: **콘텐츠 발행·편집(CMS)**, 카테고리·태그 관리, 댓글 관리, 노출·배너 관리, 조회수·통계

### 6. 마켓플레이스 (양면 거래)
예시: 중고거래, 프리랜서 매칭, 재능마켓
- 사용자(수요자): 탐색·검색, 상세, 문의·채팅, 거래·결제, 리뷰
- 사용자(공급자): 등록·판매 관리, 문의 응대, 정산 확인
- 백오피스: **양측 회원 관리**, **거래 관리**, **정산**, 분쟁·신고 처리, 수수료·정책 관리, 통계

### 7. 소셜 / 네트워킹
예시: 관심사 기반 SNS, 프로필 네트워크
- 사용자: 프로필, 팔로우/친구, 피드, 게시·공유, 메시지/DM, 알림, 검색·추천
- 백오피스: 회원 관리, **콘텐츠·신고 관리**, 추천 로직 설정, 통계

### 8. 교육 / LMS
예시: 온라인 강의, 학습 플랫폼
- 사용자: 강의 목록·상세, 수강신청·결제, 강의 수강(플레이어), 진도·수료, 퀴즈·과제, 수강 후기
- 백오피스: **강의·커리큘럼 관리**, **수강생 관리**, 결제·정산, 과제·성적 관리, 통계

### 9. 랜딩 / 소개형 사이트
예시: 회사 소개, 서비스 소개, 캠페인 페이지
- 사용자: 홈, 소개, 서비스·제품 소개, **문의/상담 신청**, 뉴스·블로그, 채용
- 백오피스: **페이지·콘텐츠 관리(CMS)**, 문의 관리, 배너·SEO 설정

### 10. 모임 / 이벤트
예시: 소모임, 번개, 원데이 클래스·세미나 참여, 동호회 정기 모임
- 사용자: 모임 목록·상세, **모임 만들기**(시간·장소·정원·설명), 참여 신청(승인제/선착순), 참여자 목록, 모임 채팅, 위치·지도, 알림(확정·시작 임박), 후기·평점, 신고
- 백오피스: **모임 관리**(블라인드/삭제), **신고 처리**, 회원 관리, 카테고리 관리, 통계
- 참고: 유료 모임이면 결제·정산을 추가한다. 위치 기반이면 위치정보 동의를 넣는다.

### 11. 온디맨드 / 호출·배달
예시: 음식 배달, 심부름·대행, 방문 청소·수리, 차량 호출
- 사용자(주문자): 위치 설정, 대상 탐색·요청, **실시간 상태 추적**, 결제, 채팅·연락, 리뷰
- 사용자(수행자·기사): 요청 수락/거절, 진행 상태 업데이트, 정산 확인
- 백오피스: **주문·배차 관리**, **수행자 관리·승인**, **정산**, 실시간 모니터링, 분쟁·CS 관리, 통계
- 참고: 실시간성과 위치가 핵심. 위치정보 동의와 결제·정산을 기본으로 넣는다.

### 12. 개인 기록 / 트래커
예시: 가계부, 습관 관리, 일기, 운동·식단 기록
- 사용자: 기록 입력, 목록·달력·차트로 보기, 통계·리포트, 알림·리마인더, 데이터 내보내기·백업
- 백오피스: (개인용이라 최소) 회원 관리, 문의 관리, 전체 통계 — **관리자 최소본**으로 가볍게
- 참고: 타인과의 상호작용이 거의 없다. **신고·커뮤니티·채팅 같은 소셜 기능을 자동으로 넣지 않는다.** 대신 데이터 보기·통계·백업을 충실히.

> 대부분의 서비스는 하나의 유형에 딱 맞지 않는다. 예: "예약도 되는 커뮤니티"는 3 + 1을 합친다.
> 핵심 유형 하나를 기준으로 잡고, 사용자가 말한 핵심 기능이 다른 유형 쪽이면 그쪽 기능도 끌어와 얹는다.

## 백오피스 짝맞춤 원칙

프론트에 어떤 기능이 생기면, 그것을 **만들고·보고·관리·정산·처리**할 사람이 필요하다.
아래 짝을 기본으로 삼아 자동으로 관리자 기능을 만든다.

| 프론트에서 생기는 것 | 관리자에 필요한 것 |
|---|---|
| 회원이 가입한다 | 회원 관리(조회·정지·탈퇴) |
| 사용자가 콘텐츠·글을 올린다 | 콘텐츠 관리·블라인드·삭제 |
| 사용자가 신고한다 | 신고 처리 큐 |
| 결제·거래가 일어난다 | 주문/거래 관리 + 정산 + 환불 |
| 문의를 남긴다 | 문의(CS) 답변 관리 |
| 공지·약관·FAQ를 본다 | 그 글을 쓰는 편집 화면 |
| 숫자가 쌓인다(가입·매출 등) | 대시보드·통계 |
| 여러 명이 관리한다 | 관리자 계정·권한 관리 |

## 화면 도출 & 네비게이션 힌트 (초안용)

- **기능 하나가 화면 하나는 아니다.** 목록형 기능은 보통 "목록 화면 + 상세 화면 + 쓰기/편집 화면" 세 개로 갈린다.
- **홈은 진입 허브**다. 핵심 행동으로 가는 길을 홈에 배치한다.
- 네비게이션 기본형: **폰 앱**은 하단 탭바(3~5개), **웹**은 상단 메뉴 + (많으면) 사이드 메뉴. 관리자 화면은 대개 **좌측 사이드 메뉴**.
- 탭/메뉴 개수는 5개 이하로 유지하고, 나머지는 "더보기"나 마이페이지 안으로 넣는다.
- 라벨은 사용자 말로. "대시보드" 대신 "홈", "엔티티 관리" 대신 "회원 관리"처럼 쉬운 이름을 쓴다.
- 상세 네비게이션 원칙(상태·역할별 메뉴, GNB/Footer, 백오피스 LNB, 이동 규칙)은 `references/네비게이션-설계-원칙.md` 참고.
