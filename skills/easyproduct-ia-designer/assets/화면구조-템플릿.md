> 이 문서는 "이 서비스가 **어떤 기능과 화면으로 이루어지는지**"를 사람이 읽기 쉽게 정리한 것입니다.

#### `FEAT.도메인.기능명` — [기능 이름]
- **무엇을 하나**: 이 기능이 하는 일을 2~3문장으로.

## 4. 화면 정의 (하나씩 자세히)
**여기가 이 문서의 알맹이다.** 주요 화면마다 아래 항목을 채워, 만들 사람이 화면을 그릴 수 있게 한다.
(모든 화면을 다 쓸 필요는 없다. 핵심 화면과 관리자 주요 화면 위주로.)

### [화면 이름]
- **대응 기능(ID)**: 이 화면이 구현하는 기능 ID. 여러 개면 여러 개 적는다. (예: `FEAT.member.search`) — IA에서 확정된 ID를 그대로 쓴다.
- **로그인 필요**: `필수` / `무관(콘텐츠 분리)` / `무관(동일)` 중 하나로 판단해 적는다. (이동 구조 5번의 "메뉴 노출 조건"과는 **다른 문제** — 여기선 화면 자체에 못 들어오는지/들어오되 내용만 갈리는지/완전히 같은지를 판단한다.)
  - `필수`: 화면이 다루는 정보가 개인 소유물(마이페이지·주문내역·내 활동 등) → 로그인 없이는 진입 불가.
  - `무관(콘텐츠 분리)`: 공개 정보를 보여주되 로그인 시 개인화 요소(내 활동·찜 등)가 섞임 → 진입은 되고 내용만 갈림.
  - `무관(동일)`: 약관·소개 등 완전 공개 정보 → 로그인 여부와 무관하게 동일.
  - 초안 생성 시 **자동으로 판단해 채우고**, 아래 "자동으로 넣은 것"에 표시해 사용자가 보고 고치기 쉽게 한다.
- **목적**: 이 화면이 왜 있고 무엇을 위한 곳인지 한 문장.
- **진입 경로**: 어디서 이 화면으로 들어오나. (예: 홈의 모임 카드를 누르면)
- **화면에 보이는 것**: 주요 구성요소·정보를 목록으로.
  - …
  - …
- **여기서 할 수 있는 일**: 버튼·동작마다 "누르면 → 무슨 일이 일어나는지".
  - [버튼/동작] → [결과]
  - …
- **상태·예외**: 상황별로 화면이 어떻게 달라지나.
  - 비어 있을 때(내용이 아직 없음): …
  - 오류일 때(불러오기 실패, 인터넷 끊김): …
  - 권한이 없을 때(로그인 안 함 등): …
- **이어지는 화면**: 여기서 갈 수 있는 다음 화면들.

## 5. 이동 구조 (네비게이션)
사용자가 화면 사이를 어떻게 오가는지. **메뉴·이동 규칙만** 정의한다(배치·고정·반응형·상태는 화면 설계서 몫).
상세 원칙은 `references/네비게이션-설계-원칙.md`를 따른다.

메뉴는 뭉뚱그리지 말고 **상태·역할별로 나눠, "항목 → 목적지(FEAT ID) → 노출 조건" 형식**으로 적는다.
목적지는 이름이 아니라 그 화면이 속한 기능의 `FEAT` ID로 가리킨다.

### 프론트 — 계층
공개 홈 → (해당 시) 서브 홈/카테고리 → 기능·상세. (로그인 필수 서비스면 공개 진입 화면과 로그인 후 홈을 구분)

### 프론트 — GNB (상단, 핵심·자주 쓰는 메뉴)
| 항목 | 목적지(FEAT ID) | 노출 조건 |
|---|---|---|
| 홈 | `FEAT.home.main` | 항상 |
| [핵심 메뉴] | `FEAT.도메인.기능명` | 항상 / 로그인 시 / [역할]일 때 |
| 마이페이지 | `FEAT.my.home` | 로그인 시 |

### 프론트 — Footer (보조·정보성만)
- 고객센터 `FEAT.support.inquiry` · FAQ `FEAT.support.faq` · 공지 `FEAT.notice.board` · 약관 `FEAT.legal.terms` · 개인정보처리방침 `FEAT.legal.privacy` · 사업자 정보
- (핵심 기능은 GNB에만. 고객센터 같은 접근성 목적의 의도적 중복만 예외)

### 관리자 — LNB (좌측 메뉴, 그룹화 + 역할별 노출)
계층: 대시보드(홈) → 목록 → 상세 → 상세 내 탭·액션.
| 그룹 | 항목 | 목적지(FEAT ID) | 노출 조건(역할) |
|---|---|---|---|
| 운영 | 회원 관리 | `FEAT.admin.member` | 관리자 |
| 운영 | [대상] 관리 | `FEAT.admin.[대상]` | 관리자 / [역할] |
| 설정 | 권한 관리 | `FEAT.admin.role` | 최고관리자 |
- (선택) 유틸리티 바: 버전·환경·도움말. 사업자 정보 등 대외 표시는 프론트가 담당하므로 반복하지 않는다.

### 이동 규칙
- 로고 = 홈(상태별). 대표 항목명 = 해당 소개/진입 화면. 브레드크럼으로 상위 단계 복귀.
- 데이터가 많은 목록(관리자 등)은 전역 검색으로 목록을 건너뛰고 상세로 직진입. 데이터가 적으면 불필요.
