# 유스케이스: [프로젝트 이름]

> 이 문서는 "사용자가 무언가를 이루려 할 때, 처음부터 끝까지 어떻게 진행되는지"를
> **목표 단위**로 정리한 것입니다. 화면 하나가 아니라, 하나의 목표를 이루는 전체 과정을 봅니다.
> 화면·기능 구조 문서에서 뽑아 재조립했고, 비어 있던 부분은 합리적으로 가정해 "가정"에 표시했습니다.
>
> 용어 풀이: **행위자** = 이 일을 하는 사람 · **사전조건** = 시작 전에 필요한 것 ·
> **기본 흐름** = 문제없이 잘 풀릴 때의 순서 · **대안 흐름** = 다른 방식으로도 되는 경우 ·
> **예외 흐름** = 문제가 생겼을 때 · **사후조건** = 끝나고 나면 어떤 상태가 되는지.
