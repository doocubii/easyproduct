# 버전·릴리즈 규칙

이 저장소(easyproduct 스킬 세트)의 버전과 릴리즈 날짜를 **언제 확정하는지** 정한 규칙이다.

## 한눈에

| 항목 | 언제 확정되나 | 어디에 기록되나 |
|---|---|---|
| **버전 번호** | 작업 브랜치가 **`beta`에 머지될 때** | `CHANGELOG.md`, 각 스킬 README, 루트 README |
| **릴리즈 날짜** | `beta`가 **`main`에 머지될 때**(정식 릴리즈) | `CHANGELOG.md`, 루트 README |

## 원칙

- **세트 단위 버전.** 스킬들은 서로 맞물려 함께 배포되므로 **하나의 세트 버전**을 공유한다
  (스킬마다 따로 버전을 매기지 않는다). 개별 스킬 README도 이 세트 버전을 표시한다.
- **버전 형식**은 `MAJOR.MINOR.PATCH`(유의적 버전, SemVer)를 따른다.
  - MAJOR — 문서 구조·ID 체계 등 **호환이 깨지는** 변경.
  - MINOR — 새 스킬·새 산출물 등 **기능 추가**(하위 호환).
  - PATCH — 문구·버그·작은 보완.
- **1.0.0 이전(0.x)** 은 아직 구조가 굳지 않은 초기 단계로 본다. 0.x에서는 MINOR가 구조 변경을 포함할 수 있다.

## 흐름 (이 저장소의 브랜치 전략과 맞물림)

```text
feature/*  →(머지)→  beta  →(머지)→  main
              │                 │
              │                 └ 릴리즈 날짜 확정 (정식 배포)
              └ 버전 번호 확정 (다음 버전으로 확정/부여)
```

1. **작업은 `beta`에서 딴 브랜치**(`feature/*`)에서 한다.
2. 작업 브랜치를 **`beta`에 머지할 때** 이번 변경에 맞는 **버전 번호를 확정**한다
   (`CHANGELOG.md`의 `[Unreleased]`를 새 버전 번호로 승격, 날짜는 아직 `미정`).
3. `beta`를 **`main`에 머지할 때(정식 릴리즈)** 그 버전의 **릴리즈 날짜를 확정**한다
   (`CHANGELOG.md`의 해당 버전에 날짜 기입).

## 현재 상태

- **다음 버전(예정): `0.1.0`** — 아직 `beta`에 머지되지 않아 **미확정(잠정)** 이다.
- **릴리즈 날짜: 미정** — `main` 머지 시 확정된다.
- 자세한 변경 내역은 `CHANGELOG.md` 참조.

> 이 잠정 버전은 `beta` 머지 시점에 확정된다. 그 전까지 각 문서에 적힌 `0.1.0`은 "예정"으로 읽는다.
