# 버전·릴리즈 규칙

이 저장소(easyproduct 스킬 세트)의 버전과 릴리즈 날짜를 **언제 확정하는지** 정한 규칙이다.

## 한눈에

| 항목 | 언제 기록되나 | 어디에 기록되나 |
|---|---|---|
| **버전 번호** | **`beta`에 변경이 들어갈 때마다** 올림(무조건) | `CHANGELOG.md`, 각 스킬 README/SKILL, 루트 README |
| **릴리즈 날짜**(단일 필드) | **머지될 때마다** 기록·갱신 — **`beta` 브랜치에선 beta 머지일, `main` 브랜치에선 main 머지일** | `CHANGELOG.md`, 루트 README, 각 스킬 SKILL |

> 릴리즈 날짜는 **두 개가 아니라 하나의 필드**다. 그 값의 뜻이 브랜치에 따라 다를 뿐 — `beta`에선 "이 베타가 나온 날", `main`에선 "정식 릴리즈된 날". `main` 머지 때 같은 필드를 그 날짜로 갱신한다.

## 원칙

- **세트 단위 버전.** 스킬들은 서로 맞물려 함께 배포되므로 **하나의 세트 버전**을 공유한다
  (스킬마다 따로 버전을 매기지 않는다). 개별 스킬 README도 이 세트 버전을 표시한다.
- **버전 형식**은 `MAJOR.MINOR.PATCH`(유의적 버전, SemVer)를 따른다.
  - MAJOR — 문서 구조·ID 체계 등 **호환이 깨지는** 변경.
  - MINOR — 새 스킬·새 산출물 등 **기능 추가**(하위 호환).
  - PATCH — 문구·버그·작은 보완.
- **1.0.0 이전(0.x)** 은 아직 구조가 굳지 않은 초기 단계로 본다. 0.x에서는 MINOR가 구조 변경을 포함할 수 있다.
- **`beta`에 변경이 들어갈 때마다 버전을 올린다(무조건).** feature 브랜치 머지든 `beta` 직접 커밋 배치든,
  `beta`에 새 변경이 랜딩되면 **반드시 버전을 올린다** — 같은 버전(`x.y.z`)으로 두 번 랜딩하지 않는다.
  변경 성격에 따라 MINOR(기능)·PATCH(수정·문서)를 고른다. 실수로 안 올렸으면 다음 랜딩에서라도 올린다.
- **공개·자기보고 표기는 스테이지 중립(중요·재발 방지).** 루트 `README.md`와 각 스킬 `SKILL.md` 메타에는
  **버전·날짜 값만** 적는다. `beta`/`main` 같은 **스테이지 라벨이나 "~머지로 확정", "현재 beta 브랜치 기준" 같은
  문구를 넣지 않는다** — 베타가 배포되든 정식이 배포되든 이 문서들은 **똑같이** 보여야 한다. 스테이지 구분과
  "언제 갱신되는지"의 규칙은 **오직 이 `VERSIONING.md`에만** 둔다. (릴리즈 시 이 규칙 위반이 반복되지 않도록
  아래 체크리스트에서 반드시 확인한다. `CHANGELOG.md`의 `(beta)` 표기는 변경 이력 기록이라 예외.)

## 흐름 (이 저장소의 브랜치 전략과 맞물림)

```text
feature/*  →(머지)→  beta  →(머지)→  main
              │                 │
              │                 └ 릴리즈 날짜 확정 (정식 배포)
              └ 버전 번호 확정 (다음 버전으로 확정/부여)
```

1. **작업은 `beta`에서 딴 브랜치**(`feature/*`)에서 한다.
2. 작업 브랜치를 **`beta`에 머지할 때** 이번 변경에 맞는 **버전 번호를 확정**하고, **릴리즈 날짜에 그 beta 머지일을 기록**한다
   (`CHANGELOG.md`의 `[Unreleased]`를 새 버전 번호 + beta 날짜로 승격).
3. `beta`를 **`main`에 머지할 때(정식 릴리즈)** 같은 **릴리즈 날짜 필드를 그 main 머지일로 갱신**한다
   (버전 번호는 그대로, 날짜만 정식 릴리즈일로 바뀐다).

## 각 스킬이 자기 버전을 안다 (자기 보고)

버전·릴리즈·배포처·라이선스는 **각 스킬의 `SKILL.md` 안(끝의 "메타 정보" 절)에도** 들어 있다.
스킬이 단독으로 설치돼도(예: claude.ai에 스킬 하나만 업로드) 사용자가 "이 스킬 버전/라이선스가 뭐야?"라고
물으면 스킬이 그 값으로 답할 수 있게 하기 위함이다. `README.md`·`VERSIONING.md`·`CHANGELOG.md`는
사람이 보는 문서일 뿐 스킬 실행 시 로드되지 않으므로, **자기 보고에 필요한 값은 `SKILL.md`가 직접 품는다.**

그 대가로 버전 문자열이 여러 곳에 존재하므로, 확정 시 **아래 체크리스트**대로 한꺼번에 갱신한다.

### `beta` 머지 시 갱신할 곳 (버전 번호 확정 + 릴리즈 날짜 = beta 머지일)

- [ ] **스킬 부합 게이트 (선행·차단 조건)** — 스킬들이 점검 원칙대로 만들어졌는지 점검한다.
      `skills/easyproduct-suite/references/checker-guide.md`의 **"스킬 부합 점검" 체크리스트**대로(빈 껍데기
      금지·뜻 필수·충실 미러 지침·SW 3종·정합성 점검 절 일치). **하나라도 실패하면 버전업/머지를 중단하고
      먼저 스킬을 고친다.** (기계 가능 항목은 스킬-린트로, 의미 판단 항목은 에이전트가.)
- [ ] `CHANGELOG.md` — `[Unreleased]`/`[예정]` 섹션을 확정 버전 번호 + beta 머지일로 승격.
- [ ] `VERSIONING.md` — 아래 "현재 상태"의 버전·날짜.
- [ ] 루트 `README.md` — "버전·릴리즈" 섹션의 버전·날짜. **값만 — 스테이지 라벨·문구 금지**(위 원칙).
- [ ] 각 스킬 `skills/*/README.md` — "버전" 항목(8곳). **값만**.
- [ ] 각 스킬 `skills/*/SKILL.md` — 끝의 "메타 정보" 절 버전·릴리즈 날짜(8곳). **값만 — 스테이지 라벨·문구 금지**.

### `main` 머지 시 갱신할 곳 (릴리즈 날짜만 → main 머지일)

- [ ] `CHANGELOG.md` · `VERSIONING.md` · 루트 `README.md` — 릴리즈 날짜를 main 머지일로 갱신.
- [ ] 각 스킬 `SKILL.md`의 "메타 정보" 절 릴리즈 날짜(8곳).

> 값이 하나(세트 전체 공유)이므로 문자열 치환으로 일괄 갱신하면 된다.
